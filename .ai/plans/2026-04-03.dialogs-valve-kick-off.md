# Plan: Build `@dialogs-valve/react` NPM library

Create a **router-agnostic**, **generic** React library published as `@dialogs-valve/react`. The library lets consumers register dialog components (drawers, modals, etc.) and control their open/close state via URL query params вЂ” making dialogs globally accessible, deep-linkable, and well-scoped. Built with **tsdown**, tested with **Vitest**, shipped as ESM + CJS with full TypeScript declarations.

---

## Architecture Overview

The library exposes:

1. **`<DialogsValveProvider>`** вЂ” Context provider at app root. Accepts a router adapter, dialog map, optional guard context, and config.
2. **`buildDialogUrl()` / `buildCloseDialogUrl()` / `buildCloseAllDialogsUrl()`** вЂ” Pure URL-building utilities (no `window.location` dependency).
3. **`useDialogsValve()`** вЂ” Hook for imperative open/close/query from within React.
4. **Types** вЂ” `DialogMap`, `DialogEntry`, `RouterAdapter`, `DialogPropValue`, etc.

## Steps

### Phase 1: Project scaffolding

1. **Initialize `package.json`** вЂ” name `@dialogs-valve/react`, `peerDependencies` on `react >=18` and `react-dom >=18`. No router dependency.
2. **Configure tsdown** вЂ” `tsdown.config.ts` targeting ESM + CJS + `.d.ts`. Entry: `src/index.ts`.
3. **Configure Vitest** вЂ” `vitest.config.ts` with jsdom environment + React Testing Library.
4. **Configure TypeScript** вЂ” `tsconfig.json`, strict mode, JSX react-jsx.
5. **Add linting/formatting** вЂ” ESLint + Prettier (lightweight).
6. **Add `.gitignore`** + `files` field in package.json for publish.

### Phase 2: Core types & constants

7. **Create `src/constants.ts`** вЂ” Define `DIALOG_MAIN_KEY` (the default query param key used to identify the active dialog) and `DIALOG_DELAY_TO_CLOSE` (milliseconds to wait before unmounting a dialog, allowing close animations to play). Both values are configurable via provider options.
8. **Create `src/types.ts`** вЂ” All public types:
   - `DialogPropValue = string | number | boolean`
   - `RouterAdapter = { navigate(url: string): void }` вЂ” pathname and search are read from `window.location` internally
   - `DialogEntry<TPermissions> = { Component: ComponentType<any>; canShow?: (ctx: TPermissions) => boolean }`
   - `DialogMap<TPermissions> = Record<string, DialogEntry<TPermissions>>`
   - `DialogsValveConfig = { dialogParamKey?: string; closeDelay?: number }`
   - `BuildDialogUrlOptions = { props?: Record<string, DialogPropValue>; overlap?: boolean; pathName?: string }`

### Phase 3: Pure utility functions (no React)

9. **Create `src/url-builder.ts`** вЂ” Functions that build URLs for opening and closing dialogs. They read `window.location.search` and `window.location.pathname` internally. Signatures: `buildDialogUrl(dialogKey, options?)`, `buildCloseDialogUrl(dialogKey)`, `buildCloseAllDialogsUrl()`.
10. **Create `src/query-params-utils.ts`** вЂ” Helper utilities for parsing and manipulating dialog-related query params: `extractSpecificProps()`, `cleanUpQueryParams()`, `parsePropValueWithCorrectType()`, `validateDialogKeys(keys, validKeys)` (takes valid keys as a param to stay decoupled from any specific dialog registry).

### Phase 4: React layer

11. **Create `src/context.ts`** вЂ” `DialogsValveContext` via `React.createContext`.
12. **Create `src/provider.tsx`** вЂ” `<DialogsValveProvider>` component:
    - Props: `router: RouterAdapter`, `dialogs: DialogMap<TPermissions>`, `permissions?: TPermissions`, `config?: DialogsValveConfig`, `children`
    - Reads `window.location.search` to derive `currentDialogKeys`
    - Manages the delayed-close state machine (`isDOMClean` / `currentDialogKeysWithDelay` logic)
    - Renders dialog components, validates keys, calls `canShow`, emits `console.warn` on unauthorized
13. **Create `src/hooks.ts`** вЂ” `useDialogsValve()` hook returning:
    - `openDialog(key, options?)`, `closeDialog(key)`, `closeAllDialogs()` вЂ” imperative API calling `router.navigate`
    - `isOpen(key)`, `getDialogProps(key)` вЂ” read state
14. **Create `src/index.ts`** вЂ” Barrel export of all public API.

### Phase 5: Tests

15. **Create `src/__tests__/url-builder.test.ts`** вЂ” Cover: basic URL construction, props encoding (string/bool/number), overlap on/off, close single dialog, close all dialogs.
16. **Create `src/__tests__/query-params-utils.test.ts`** вЂ” Cover: prop extraction, cleanup, key validation, type parsing.
17. **Create `src/__tests__/provider.test.tsx`** вЂ” Integration tests with React Testing Library + mock `RouterAdapter`: renders dialog on matching param, passes extracted props, guard rejection + console.warn, delayed close, overlap, onClose navigation.
18. **Create `src/__tests__/hooks.test.tsx`** вЂ” `renderHook` tests: openDialog/closeDialog navigate correctly, isOpen returns correct state.

### Phase 6: Documentation & publish

19. **Write `README.md`** вЂ” Installation, quick start, API reference, examples (react-router-dom adapter, Next.js adapter), guard system, config options.
20. **Write `CHANGELOG.md`** вЂ” Initial v0.1.0.
21. **Configure npm publish** вЂ” `files` field, `prepublishOnly` в†’ `tsdown`, `"type": "module"`, proper `exports` field with ESM/CJS/types paths.
22. **GitHub Actions CI** (optional) вЂ” lint + test + build on PR.

## Usage Example

### 1. Define your dialog map

```tsx
// dialogs-map.ts
import { ConfirmDeleteDrawer } from "./drawers/confirm-delete-drawer";
import { UserProfileDrawer } from "./drawers/user-profile-drawer";
import type { DialogMap } from "@dialogs-valve/react";
import type { MyPermissions } from "./types";

export const DIALOGS = {
  "confirm-delete-drawer": {
    Component: ConfirmDeleteDrawer,
  },
  "user-profile-drawer": {
    Component: UserProfileDrawer,
    canShow: (permissions: MyPermissions) => permissions.canViewProfiles,
  },
} as const satisfies DialogMap<MyPermissions>;
```

### 2. Set up the provider (e.g. with react-router-dom)

```tsx
// app.tsx
import { DialogsValveProvider } from "@dialogs-valve/react";
import { useNavigate } from "react-router-dom";
import { DIALOGS } from "./dialogs-map";
import { useMyPermissions } from "./hooks";

function App() {
  const navigate = useNavigate();
  const permissions = useMyPermissions();

  return (
    <DialogsValveProvider
      router={{ navigate }}
      dialogs={DIALOGS}
      permissions={permissions}
    >
      <RestOfYourApp />
    </DialogsValveProvider>
  );
}
```

### 3. Open / close dialogs from anywhere

```tsx
// some-component.tsx
import { useDialogsValve } from "@dialogs-valve/react";

function SomeComponent() {
  const { openDialog, closeDialog } = useDialogsValve();

  return (
    <button
      onClick={() =>
        openDialog("user-profile-drawer", {
          props: { userId: "42", readOnly: true },
          overlap: true,
        })
      }
    >
      View Profile
    </button>
  );
}
```

### 4. Dialog component contract

```tsx
// drawers/user-profile-drawer.tsx
type Props = {
  open: boolean; // controlled by the library
  onClose: () => void; // controlled by the library
  userId?: string; // custom prop passed via query params (optional)
  readOnly?: boolean; // custom prop passed via query params (optional)
};

export function UserProfileDrawer({ open, onClose, userId, readOnly }: Props) {
  // your drawer/modal UI here
}
```

## Decisions

- **Router-agnostic** via `RouterAdapter` вЂ” no react-router-dom dependency
- **Generic guard** вЂ” `canShow<TPermissions>`, context via provider prop
- **Console.warn only** on unauthorized вЂ” no error notification in the library
- **`@dialogs-valve/react`** scoped package name
- **tsdown** build, **Vitest** tests
- **`buildDialogUrl` reads `window.location`** internally for search + pathname; `useDialogsValve()` wraps it for convenience
- **Type-safe keys** вЂ” consumers use `as const satisfies DialogMap<T>` for autocomplete
- **Permissions are fully generic** via `TPermissions` вЂ” no built-in permissions model
- **`DIALOG_MAIN_KEY` and `DIALOG_DELAY_TO_CLOSE` are configurable** via provider config, with sensible defaults

---

## Further Considerations

1. **SSR support?** вЂ” Since the library reads `window.location` directly, the provider should gracefully handle missing `window` and return `null` during SSR. Low effort, high value.
2. **Monorepo extensibility** вЂ” The `@dialogs-valve` scope sets you up for future packages like `@dialogs-valve/next` or `@dialogs-valve/solid` if needed.
